### Como foi calculada

A prescrição combina três sinais do próprio equipamento. As fórmulas (de `s08_prescricao_manutencao.py`):

```text
T_base          = mediana(dias_em_operacao do histórico do equipamento)
fator_desgaste  = clamp( média( T_base/amp_atual_cil , T_base/amp_atual_fuso ) ,
                         0.60 , 1.20 )           # baseado nas amplitudes
                  → forma usada: 1 / (amp_atual / amp_mediana_hist)
fator_massa     = clamp( 1 / (massa_atual / massa_mediana_janelas_anteriores) ,
                         0.70 , 1.30 )
T_prescrito     = T_base × fator_desgaste × fator_massa
data_prescrita  = data_ultima_sub + T_prescrito + dias_ociosidade
```

| **Amplitude cil./fuso** | cil. 0,020 / fuso 0,290 | cil. 0,020 / fuso 0,290 | 1,0000 | neutro / sem dado |
| **Massa consumida pós-última troca (kg)** | 1.161,16 | mediana janelas anteriores: 1.626,68 | 1,3000 | consumiu **0,71×** a massa típica → adia a próxima troca (×1,300) |

### Substituindo na fórmula

```text
T_prescrito = 490,0 × 1,0000 × 1,3000 = 637,0 dias
data_prescrita = 05/07/2025 + 637 dias + 27 dia(s) ociosidade = 30/04/2027
```

### Por que a urgência ficou **OK**

- `dias_restantes < 0`  → **ATRASADO** (data prescrita já passou)
- `0 ≤ dias_restantes < 30`  → **URGENTE**
- `30 ≤ dias_restantes < 90`  → **ATENÇÃO**
- `dias_restantes ≥ 90`  → **OK**

Para este equipamento, `dias_restantes = 368` ⇒ **OK**.

### Observações sobre a fórmula

- `fator_desgaste` é limitado ao intervalo **[0,60 ; 1,20]** para evitar que outliers em históricos curtos puxem a data demais.
- `fator_massa` é limitado ao intervalo **[0,70 ; 1,30]** pelo mesmo motivo.
- Quando não há leitura histórica suficiente, o fator vira **1,00** (neutro) — o sinal correspondente é silenciado, não chuta.
- `dias_ociosidade` só **adiciona** prazo (máquina parada não desgasta); valores negativos são tratados como zero.
